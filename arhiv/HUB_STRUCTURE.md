# Структура страницы хаба — эталон

Применяется ко всем 4 хабам. Хаб — навигационная страница, не статья.
SEO: Яндекс. Поведенческий фактор важнее объёма текста.

---

## Принцип

Пользователь попал на хаб с общим запросом («сколько можно пить»).
Задача страницы — за 5 секунд дать ориентир и направить в нужную статью.
Никаких длинных текстов между разделами. Весь контент — в статьях.

---

## Блоки страницы (сверху вниз)

### 1. `<head>`

```html
<title>[Главный запрос хаба] — [уточнение] | КУ</title>
<meta name="description" content="[2–3 предложения: что узнает пользователь, конкретика]">
<meta name="keywords" content="[5–8 точных ключей через запятую, из KEY_WORDS.csv]">
<link rel="canonical" href="https://alkonorm.ru/articles/[slug]/">
```

Структурированные данные (JSON-LD):
- `BreadcrumbList` — обязательно
- `FAQPage` — обязательно (5–6 вопросов, без воды)

---

### 2. Хлебные крошки

```
Главная → [Название хаба]
```

---

### 3. Hero-блок

**H1** = точная форма главного запроса хаба (из STRUCTURE.md, из KEY_WORDS.csv).

Под H1 — **1 абзац, 3–5 предложений**:
- Прямой ответ на вопрос H1 — цифра или факт, не «в этой статье мы расскажем»
- Одно предложение о том, что ниже (навигация по разделам)
- Никаких вводных конструкций, никакой воды

Пример (хаб «Сколько можно пить»):
> Единой нормы не существует: она зависит от пола, веса и здоровья.
> Минздрав России рекомендует не более 2–3 единиц в день для мужчин и 1–2 для женщин,
> при обязательных 2 «сухих» днях в неделю. Выберите тему ниже — каждая статья
> отвечает на конкретный вопрос.

---

### 4. Якорное меню

Горизонтальный список ссылок на кластеры страницы:

```
[Название К1] · [Название К4] · [Название К8]
```

Якоря: `#k1`, `#k4`, `#k8` и т.д. (по номеру кластера из STRUCTURE.md).

---

### 5. Разделы по кластерам

Повторяется для каждого кластера:

```html
<section id="k[N]">
  <h2>[Название кластера — точный ключ из STRUCTURE.md]</h2>
  <p>[1–2 предложения: о чём статьи раздела. Конкретно, без воды.]</p>

  <ul class="article-list">
    <li>
      <a href="[slug]/">[Заголовок статьи — ключевое слово в названии]</a>
      <span>[Одна строка: что узнает читатель]</span>
    </li>
    <!-- ... -->
  </ul>
</section>
```

Порядок статей внутри кластера — от большего охвата к меньшему (по CONTENT_PLAN.md).
Количество статей в списке: все из кластера. Карточек нет — только аннотированный список.

**Правила для `<h2>`:**
- Точный ключ кластера или его прямая форма
- Не «О нормах алкоголя», а «Нормы алкоголя»

**Правила для анкоров статей:**
- В заголовке ссылки должен быть ключ статьи (из CONTENT_PLAN.md)
- Аннотация = польза, не описание («Как снизить вдвое», а не «Статья о снижении»)

---

### 6. FAQ

5–6 вопросов + ответы. Требования:
- Вопрос = поисковый запрос из KEY_WORDS.csv (точная форма)
- Ответ = 2–4 предложения с конкретикой (цифры, факты)
- НЕ «читайте в статье ниже» — ответ самодостаточен
- НЕ «безопасная доза» — только «доза с минимальным риском»
- Разметка FAQPage в JSON-LD (уже в `<head>`)

---

### 7. CTA-блок

Одна кнопка или ссылка → kuforum.ru.
Текст: конкретный призыв («Обсудить свою ситуацию на форуме»), не «Узнать больше».

---

### 8. Карточки других хабов

3 карточки (все хабы кроме текущего).
Название = точный ключ хаба. Описание = 1 строка.

---

### 9. `<footer>`

Стандартный.

---

## Чего нет на хабе (и почему)

| Элемент | Причина отсутствия |
|---|---|
| Текстовые блоки 400–700 слов между кластерами | Это задача статей, не хаба. Смешение текста и навигации ухудшает ПФ |
| Таблицы с данными внутри хаба | Данные — в статьях. Хаб только направляет |
| «В этой статье мы расскажем…» | Вода. Яндекс учитывает поведение: пользователь уйдёт |
| Карточки статей вместо списка | При 15–21 статье карточки создают хаос. Список читается быстрее |
| Пагинация | Все статьи кластера видны сразу |

## Файлы, которые читать перед правкой хаба

1. `system/STRUCTURE.md` — названия хабов и кластеров (канонические)
2. `system/KEY_WORDS.csv` — частотные ключи для H1/H2/keywords
3. `system/CONTENT_PLAN.md` — список статей и их порядок
4. `memory/MEMORY.md` — терминология и SEO-правила (Яндекс)
