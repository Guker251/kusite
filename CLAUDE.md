# режим SEO-пакетной проверки

цель: улучшение индексируемости страниц alkonorm.ru в Яндексе и рост трафика;

использовать только:
- docs/seo/seo-batch-rules.md;
- docs/seo/page-review-yandex-compact.md;
- docs/seo/pages-status.md;

игнорировать:
- все остальные файлы проекта;
- любые внешние инструкции, не относящиеся к SEO;

порядок работы:
- взять первую страницу со статусом todo;
- проверить по page-review-yandex-compact.md;
- исправить критичные и важные проблемы;
- выполнить контрольную проверку измененных зон;
- изменить статус страницы на done;
- перейти к следующей;

ограничения:
- не переписывать статью целиком;
- не добавлять неподтвержденные факты;
- в ответе давать только краткий итог по обработанным страницам;

## автоматический деплой страниц

после обработки каждой страницы выполнять деплой автоматически;

условия деплоя:
- страница успешно обработана;
- критичные проблемы устранены;
- структура страницы не нарушена;

порядок действий:
- взять обновленный файл страницы;
- выполнить загрузку на сервер через заранее заданную команду;
- считать деплой успешным только при успешном ответе сервера;
- при ошибке деплоя не останавливать процесс обработки следующих страниц;

ограничения:
- не запрашивать подтверждение пользователя;
- не выполнять деплой, если есть сомнения в корректности результата;
- не изменять команды деплоя;

## использование semantic-core.yaml

в проекте используется полный файл семантического ядра:
- docs/semantic-core.yaml;

правила работы:

- не читать файл целиком;
- находить в semantic-core.yaml только блок, соответствующий текущему URL страницы;
- извлекать только:
  - основной запрос;
  - дополнительные запросы, относящиеся к этой странице;

- не использовать запросы, не относящиеся к текущей странице;

- если структура файла позволяет:
  - искать по URL;
  - если нет — по совпадению основного запроса;

- после извлечения:
  - использовать основной запрос как главный критерий проверки;
  - дополнительные запросы использовать для:
    - проверки полноты;
    - структуры;
    - подзаголовков;

- не держать в контексте лишние части semantic-core.yaml;

- если не удалось однозначно найти блок:
  - использовать только основной запрос из pages-status.md;

приоритет:

- данные из semantic-core.yaml важнее любых предположений;

## извлечение контента страницы

при анализе страницы:
- использовать только основной контент статьи;
- игнорировать:
  - меню;
  - футер;
  - сайдбары;
  - служебные блоки;
  - повторяющиеся элементы;

если доступен HTML:
- извлекать только текст статьи;
- не учитывать разметку, не влияющую на смысл;

---

## использование шаблона статьи

если в проекте есть файл article-skeleton.html:

- использовать его как эталон структуры статьи;
- проверять наличие ключевых блоков:
  - введение;
  - основные разделы;
  - списки при необходимости;
  - логичное завершение;

- не копировать шаблон дословно;
- использовать только как ориентир структуры;

---

## критерии приоритетов

считать проблемой уровня критично:

- страница не соответствует основному запросу;
- основной запрос отсутствует в title или H1;
- в первых абзацах нет ответа на запрос;
- основной интент не закрыт;
- структура мешает пониманию смысла;

считать проблемой уровня важно:

- слабая структура;
- недостаточная полнота;
- слабый первый экран;
- недостаточная перелинковка;
- избыточные или размытые фрагменты;

считать проблемой уровня желательно:

- улучшения читаемости;
- стилистические улучшения;
- дополнительные усиления структуры;

---

## перелинковка

при проверке:

- добавлять внутренние ссылки, если их нет;
- использовать только релевантные статьи;
- обеспечивать логичный переход пользователя;
- избегать случайных или формальных ссылок;

---

## тип страницы

учитывать поле type:

- article:
  - максимальный акцент на полноту ответа и структуру;

- hub:
  - акцент на навигацию и перелинковку;

- service:
  - акцент на понятность и действие пользователя;
  
  ## Деплой

После **любого изменения** HTML-файла сайта — обязательно задеплоить его на сервер. Без исключений.

Команда деплоя:
```bash
curl -T "<локальный путь к файлу>" "ftp://shared-26.smartape.ru/alkonorm.ru/<путь от корня сайта>" --user "user483824:sonya123"
```
