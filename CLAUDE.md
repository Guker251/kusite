# Цель проекта
Создание сайта по контролируемому употреблению алкоголя для привлечения участников на форум 

# Аудитория проекта
Люди, интересующиеся ограничением, контролем или соблюдением нормы употребления алкоголя.

# Правила проекта

Семантическое ядро разбито по хабам. Каждый хаб — отдельный файл в `docs/hubs/`:

| Хаб | Файл | URL |
|-----|------|-----|
| H1 — Сколько можно пить | `docs/hubs/semantic-core-H1.yaml` | `articles/skolko-mozhno-pit/` |
| H2 — Как меньше пить | `docs/hubs/semantic-core-H2.yaml` | `articles/kak-menshe-pit/` |
| H3 — Почему хочется алкоголя | `docs/hubs/semantic-core-H3.yaml` | `articles/pochemu-hochetsya-alkogolya/` |
| H4 — Признаки алкоголизма | `docs/hubs/semantic-core-H4.yaml` | `articles/uznaki-alkogolizma/` |

**Правило:** при любой задаче открывать только файл того хаба, с которым ведётся работа. Не читать файлы других хабов без необходимости.

Общий файл `docs/semantic-core.yaml` сохранён как архив. Для текущей работы не использовать — использовать только файлы по хабам.

**Новые кластеры (март 2026):** семантика в `docs/semantic-core-new-clusters.yaml`. Хаб не назначен — перед созданием статьи уточнить у пользователя, в какой хаб помещать. После назначения хаба — добавить кластер в соответствующий `docs/hubs/semantic-core-H*.yaml` и `docs/hubs/hub-*.md`.

Иерархия сущностей:
- hub;
- cluster; 
- keyword;

При работе с сайтом всегда сначала определяй:
- hub;
- cluster;
- цель задачи;

Правила:
- не придумывай новые hub без явного указания;
- не придумывай новые cluster без явного указания;
- все решения по структуре сайта принимай только на основе файла соответствующего хаба (`docs/hubs/semantic-core-H*.yaml`);
- не смешивай hub, cluster и article;
- article создается внутри существующего `hub`;
- порядок статей не должен меняться;

Пиши в ответе только вопросы пользователю по выполнению заданий и что выполнено. Не описывай процесс выполнения.

## CSS

**Никогда не добавлять `<style>` блок** в HTML-файл статьи или хаба.

### Статьи (`articles/[хаб]/[слаг]/index.html`)

Подключают только `article.css`:
```html
<link href="https://fonts.googleapis.com/css2?family=Montserrat:wght@300;400;500;600;700&family=Jost:wght@300;400;500;600&display=swap" rel="stylesheet">
<link rel="stylesheet" href="../../../assets/article.css">
```

Если нужен новый CSS-класс для статьи — добавить в `assets/article.css`.

### Хаб-страницы (`articles/[хаб]/index.html`)

Подключают оба файла:
```html
<link href="https://fonts.googleapis.com/css2?family=Montserrat:wght@300;400;500;600;700&family=Jost:wght@300;400;500;600&display=swap" rel="stylesheet">
<link rel="stylesheet" href="../../assets/article.css">
<link rel="stylesheet" href="../../assets/hub.css">
```

На теге `<body>` обязателен хаб-класс (определяет акцентный цвет):

| Хаб | Класс на `<body>` |
|-----|-------------------|
| `skolko-mozhno-pit` | *(без класса — голубой по умолчанию)* |
| `kak-menshe-pit` | `class="hub-narrow hub-green"` |
| `pochemu-hochetsya-alkogolya` | `class="hub-narrow hub-pink"` |
| `uznaki-alkogolizma` | `class="hub-narrow hub-orange"` |

Если нужен новый CSS-класс для хаба — добавить в `assets/hub.css`.

Каркас новой статьи: `docs/article-skeleton.html`.

## Деплой

После **любого изменения** HTML-файла сайта — обязательно задеплоить его на сервер. Без исключений.

Команда деплоя:
```bash
curl -T "<локальный путь к файлу>" "ftp://shared-26.smartape.ru/alkonorm.ru/<путь от корня сайта>" --user "user483824:sonya123"
```

Пример:
```bash
curl -T "/Users/guker/Desktop/ku/articles/skolko-mozhno-pit/norma-alkogolya-za-rulem/index.html" \
  "ftp://shared-26.smartape.ru/alkonorm.ru/articles/skolko-mozhno-pit/norma-alkogolya-za-rulem/index.html" \
  --user "user483824:sonya123"
```

Код ответа `226` = успех. Любой другой — сообщить пользователю.

Деплой выполняется **после** сохранения файла, **до** сообщения пользователю о завершении задачи.

## Блок-ссылка на форум (обязателен в каждой статье)

Каждая статья **обязана** содержать блок-ссылку на форум kuforum.ru. Дизайн — `cta-box` + `btn-primary` (классы в `assets/article.css`).

Расположение: перед `<section class="related-section">` или, если её нет, перед `</article>`.

Шаблон:
```html
<!-- CTA -->
<div class="cta-box">
    <h2>[Заголовок: что обсуждают на форуме по теме статьи]</h2>
    <p>[1–2 предложения: что именно можно найти/обсудить на форуме КУ по теме этой статьи]</p>
    <a href="https://kuforum.ru" target="_blank" rel="noopener" class="btn-primary">Перейти на форум →</a>
</div>
```

- Заголовок и текст должны соответствовать теме статьи
- Никогда не использовать устаревшие классы `cta-block` и `btn-ku`
- Скелет статьи (`docs/article-skeleton.html`) уже содержит этот блок

## Создание новой статьи — чеклист

При создании любой новой статьи обязательно выполнить **оба шага**:

1. Создать файл статьи (`articles/[хаб]/[слаг]/index.html`) и задеплоить.
2. Открыть хаб-страницу (`articles/[хаб]/index.html`), найти соответствующий пункт со статусом «Скоро» (`<span class="soon-title">`) и заменить на активную ссылку (`<a href="[слаг]/">`), затем задеплоить хаб.
3. Поместить статью в Карту сайта.

Шаг 2 обязателен. Без него ссылка на статью не появится на сайте.

## Книга

Материалы книги находятся в `docs/book`.

`docs/book/knowledge-map.yaml` — оглавление. Каждый топик содержит:
- `subtopics` — список заголовков секций внутри файла;
- `sources` — путь к файлу главы.

**Алгоритм работы с книгой:**

1. Прочитать `knowledge-map.yaml` целиком (файл маленький).
2. Найти нужный топик по `title` или `subtopics`. Запомнить путь из `sources`.
3. Через Grep найти строку нужного заголовка в файле главы:
   ```
   Grep pattern="<текст заголовка>" path="docs/book/<файл>.md"
   ```
4. Прочитать только нужную секцию через `offset` + `limit` в Read:
   - `offset` = номер строки заголовка − 1
   - `limit` = примерное число строк до следующего заголовка того же уровня (обычно 50–200)
5. Если нужна следующая секция — повторить Grep для её заголовка.

**Запрещено:** читать файл главы целиком через Read без offset/limit.
