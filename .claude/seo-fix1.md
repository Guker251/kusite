## Шаг 12. Минимальная схема правки title и H1

Правило:
- Title должен включать основной запрос и обещать конкретную пользу.
- H1 должен быть естественным и близким к основному запросу.
- Title и H1 могут быть похожими, но не обязаны быть дословно одинаковыми.
- Не делать title искусственным набором ключей.
- Не делать H1 слишком длинным.

Пример логики:
- Основной запрос: `как меньше пить алкоголь`
- Title: `Как меньше пить алкоголь: практический план снижения`
- H1: `Как меньше пить алкоголь`

## Шаг 13. Минимальная схема первого абзаца

Первый абзац должен:
- Прямо отвечать на основной запрос.
- Объяснять, какую задачу решает статья.
- Использовать простой язык.
- Не начинаться с общих рассуждений.
- Не содержать неподтверждённых медицинских утверждений.

Формула:
- Что человек хочет сделать.
- С чего начать.
- Что даст статья.

## Шаг 16. Отправка страницы на переобход Яндекса

Отправлять на переобход только если:
- Страница отдаёт 200.
- Страница является контентной.
- Страница не имеет `noindex`.
- Canonical указывает на саму страницу.
- Страница есть в sitemap.
- Файл успешно задеплоен.
- Опубликованная версия проверена.

Команда:

`curl -s -X POST \
"https://api.webmaster.yandex.net/v4/user/$YANDEX_WEBMASTER_USER_ID/hosts/$YANDEX_WEBMASTER_HOST_ID/recrawl/queue" \
-H "Authorization: OAuth $YANDEX_WEBMASTER_TOKEN" \
-H "Content-Type: application/json" \
-d "{\"url\": \"<URL страницы>\"}"`

Проверить ответ API:
- Если ответ успешный, записать URL в журнал.
- Если ошибка лимита, записать и не повторять бесконечно.
- Если ошибка авторизации, остановить отправку остальных URL.
- Если ошибка host-id, проверить переменную `YANDEX_WEBMASTER_HOST_ID`.
