
# единый skill проверки и исправления страницы

## цель
Проверить страницу и устранить проблемы, влияющие на индексируемость и трафик в Яндексе.

## вход
- URL;
- текст страницы;
- основной запрос;
- тип страницы;

## источник данных

- использовать semantic-core.yaml для определения запросов;
- использовать только блок, относящийся к текущему URL;

---

## очистка контента

- анализировать только основной текст статьи;
- игнорировать служебные элементы страницы;

---

## использование шаблона

если доступен article-skeleton.html:

- сравнивать структуру статьи с шаблоном;
- при необходимости:
  - добавлять недостающие структурные элементы;
  - выравнивать порядок блоков;

## зоны проверки

### 1. соответствие запросу
- есть ли запрос в title;
- есть ли запрос в H1;
- раскрыт ли запрос в начале;
- соответствует ли интенту;

### 1а. главные проверки

- проверять соответствие основному запросу в первых 2 абзацах;
- проверять полноту ответа на интент;
- запрещать любые «широкие» правки вне критичных зон;

### 2. индексируемость
- корректность title;
- наличие H1;
- отсутствие дублей;
- отсутствие переоптимизации;
- достаточность объема;

### 3. структура
- есть ли подзаголовки;
- есть ли логика блоков;
- нет ли сплошных массивов текста;

### 4. полнота
- закрыт ли основной вопрос;
- нет ли смысловых пробелов;

### 5. перелинковка
- есть ли минимум 2 релевантные внутренние ссылки;
- есть ли связь с тематическими статьями или хабом;
- есть ли логичный следующий шаг для пользователя;

### 6. поведенка
- понятен ли первый экран;
- есть ли мотивация читать дальше;

## приоритеты
- критично;
- важно;
- желательно;

## порядок работы
- выявить проблемы;
- исправить:
  - критично;
  - важно;
  - не использовать предположения, если данные не найдены в semantic-core.yaml;

- исправлять локально;
- не менять лишнего;

- после исправлений:
  - проверить только измененные зоны;
  - убедиться, что:
    - критичные проблемы устранены;
    - новые проблемы не появились;

## правила
- минимальные изменения;
- сохранение структуры;
- усиление смысла;
- без переспама;
