# Структура статьи — эталон

Применяется ко всем статьям сайта. Статья — это развёрнутый ответ на один конкретный запрос.
SEO: Яндекс. Поведенческий фактор важнее объёма текста.

---

## Принцип

Пользователь пришёл с конкретным вопросом. Он должен получить ответ в первых 3–5 предложениях.
Остальной текст — углубление, детали, нюансы для тех, кто хочет больше.
Никакой воды в начале. Никаких «в этой статье мы расскажем».

## Блоки страницы (сверху вниз)

### 1. `<head>`

```html
<title>[Точный заголовок статьи из docs/hubs] | КУ</title>
<meta name="description" content="[Прямой ответ на вопрос статьи. 1–2 предложения с цифрой или фактом. Не более 160 символов.]">
<meta name="keywords" content="[5–10 точных ключей через запятую — основной запрос + производные из docs/hubs/semantic-core-H*.yaml]">
<link rel="canonical" href="https://alkonorm.ru/articles/[хаб]/[слаг]/">
<link href="https://fonts.googleapis.com/css2?family=Montserrat:wght@300;400;500;600;700&family=Jost:wght@300;400;500;600&display=swap" rel="stylesheet">
<link rel="stylesheet" href="../../../assets/article.css">
```

**Путь к CSS зависит от глубины вложенности:**
- `articles/[хаб]/[слаг]/index.html` → `../../../assets/article.css`
- `articles/[хаб]/index.html` → `../../assets/article.css`

**❌ Не добавлять `<style>` блок в статью.** Весь CSS — только в `assets/article.css`.
Если нужен новый стиль — добавить в `assets/article.css`, а не инлайном.

Структурированные данные (JSON-LD):
- `BreadcrumbList` — всегда (Главная → Хаб → Статья)
- `FAQPage` — если в статье есть FAQ-блок (рекомендуется)
- `Article` — опционально, для новостных/экспертных материалов

### 2. Хлебные крошки

```
Главная → [Название хаба] → [Название статьи]
```

Три уровня. Ссылки кликабельны на первых двух.

### 3. Заголовок статьи (H1 + лид)

**H1** = точная форма основного запроса из hubs/
Совпадает с `<title>` или отличается на 1–2 слова.

**Лид** — 2–4 предложения сразу после H1:
- Первое предложение = прямой ответ на вопрос H1 с конкретной цифрой или фактом
- Второе — уточнение или важная оговорка
- Третье — что пользователь найдёт дальше (не «в статье», а суть)
- Никаких вводных слов: «актуальность», «в наше время», «многие задаются вопросом», "отдельная оговорка"
- Никаких отрицаний типа "это не ..., а ...";

**Пример (статья «Норма алкоголя за рулём»):**
> Допустимая норма алкоголя за рулём в России — 0,16 мг/л в выдыхаемом воздухе или 0,35‰ в крови.
> Это не «разрешение выпить»: норма введена для погрешности приборов и эндогенного алкоголя.
> Ниже — как эти цифры перевести в конкретные напитки и что происходит при превышении.

### 4. Блок быстрого ответа (опционально)

Используется когда основной запрос предполагает конкретную цифру или список.
Оформляется как выделенный блок (`.quick-answer`) сразу после лида.

```
┌─────────────────────────────────────────────┐
│ Коротко: [1–3 факта пунктами или цифрами]   │
└─────────────────────────────────────────────┘
```

Яндекс часто выбирает такие блоки для расширенного сниппета.

---

### 5. Основной контент

Структура: несколько секций с H2, внутри которых могут быть H3.

**Правила для H2:**
- Точный ключевой запрос или его прямая форма (из docs/hubs/semantic-core-H*.yaml)
- Не «Общая информация», не «Подробнее о теме» — только конкретный вопрос или утверждение
- Оптимально 2–4 H2 на статью

**Правила для H3:**
- Уточнение или подтема H2
- Можно использовать смысловые вопросы («Почему у женщин норма ниже?»)

**Объём текста:**
- Оптимально 700–1 500 слов
- Меньше 500 — Яндекс может считать страницу «тонкой»
- Больше 2 000 — риск роста отказов, если материал не удерживает

**Правила содержания:**
- Каждый абзац — одна мысль, не более 4–5 строк
- Цифры, источники, конкретные примеры — не абстракции
- Источники называть конкретно: «ВОЗ (2023)», «исследование Lancet, 2018», «Gilovich et al., 1999» — без «по данным специалистов»
- Минздрав России не упоминать
- НЕ «безопасная доза» — только «доза с минимальным риском»
- НЕ повторять H1 дословно в тексте более 2–3 раз — Яндекс видит переспам

### 6. Таблицы и списки

Используются когда данные сравниваются или перечисляются (≥ 3 элементов).

**Таблица уместна для:**
- Сравнения нормы по странам, полу, напиткам
- Таблицы промилле / степеней
- Любых «X vs Y» данных

**Список уместен для:**
- Признаков, симптомов, правил (≥ 4 пунктов)
- Пошаговых инструкций

**Правила оформления:**
- Таблица: заголовки — точные ключи, данные — конкретные цифры
- Список: каждый пункт начинается с глагола или существительного (параллельная структура)
- После таблицы/списка — 1–2 предложения интерпретации (что это значит для читателя)

---

### 7. FAQ-блок

3–5 вопросов в конце статьи. Требования:
- Вопрос = поисковый запрос из docs/hubs/semantic-core-H*.yaml (точная форма)
- Ответ = 2–4 предложения, самодостаточный (не «смотрите выше»)
- НЕ дублирует H2 статьи — это смежные и уточняющие вопросы
- Разметка FAQPage в JSON-LD (в `<head>`)

FAQ решает две задачи:
1. Добавляет длиннохвостовые запросы на страницу
2. Яндекс показывает раскрытые вопросы в расширенном сниппете

---

### 8. CTA-блок

Одна строка + ссылка → kuforum.ru.
Текст: конкретная ситуация, не «узнать больше».

Пример:
> Разбираете свою норму или хотите поговорить о конкретной ситуации? Это как раз тема форума КУ.
> [Обсудить на форуме →]

---

### 9. Блок «Читать дальше»

3–5 ссылок на другие статьи того же хаба — аннотированный список, не карточки.
Порядок: наиболее релевантные к теме статьи первыми.
Заголовок блока: «Читать по теме» или «Статьи по теме».

Ссылки на статьи других хабов — не более 1–2, только если тема пересекается.

## Чек-лист перед публикацией

| Элемент | Проверить |
|---|---|
| H1 | Содержит точный основной ключ |
| Лид | Первое предложение = прямой ответ с цифрой |
| meta description | До 160 символов, цифра или конкретный факт |
| meta keywords | 5–10 ключей, нет повторов |
| H2 | Содержат ключи из docs/hubs/semantic-core-H*.yaml |
| «безопасная доза» | Отсутствует на странице |
| `<style>` блок | Отсутствует — CSS только через `assets/article.css` |
| Источники | Названы конкретно (ВОЗ 2025, Lancet 2018) — Минздрав РФ не упоминать |
| FAQ | 3–5 вопросов, разметка FAQPage в JSON-LD |
| BreadcrumbList | 3 уровня, корректные URL |
| canonical | Указан верный URL статьи |
