## 3. Алгоритм подготовки к статье

Пользователь называет хаб и статью (по номеру или названию). Дальше:

1. **Найти статью в контент-плане:**
   - Если статья из основного плана → `docs/CONTENT_PLAN.md`
   - Если статья из новых кластеров → `docs/content-plan-new-clusters.md`
   - Взять: заголовок, основной запрос, слаг, кластер, хаб.

2. **Открыть семантику нужного хаба:**
   - Основные кластеры → `docs/hubs/semantic-core-H*.yaml`
   - Новые кластеры → `docs/semantic-core-new-clusters.yaml`
   - Выписать все ключи (пойдут в H2, FAQ, meta keywords).

3. **Открыть `docs/facts.md`** → найти цифры для лида и quick-answer.

4. **При необходимости:** прочитать `docs/book/knowledge-map.yaml` → найти нужный топик → прочитать только релевантную секцию главы (через Grep + Read с offset/limit).

5. **Предложить:** H1, структуру H2–H3, 3–5 FAQ-вопросов — и ждать подтверждения.

После подтверждения структуры: писать полный HTML по каркасу `docs/article-skeleton.html`.

## 5. Терминология — запреты

| Нельзя | Использовать вместо |
|--------|---------------------|
| «безопасная доза алкоголя» | «доза с минимальным риском» |
| Минздрав России | не упоминать вообще |

**Формула ЛЧА:**
```
ЛЧА = Объём (л) × Крепость (%) / 100
```
Объём — в литрах. Вариант с мл не использовать.

**Источники:** называть конкретно — ВОЗ (2023), Lancet (2018), Gilovich et al. (1999). Без «по данным специалистов».

**Психологические и поведенческие утверждения:** не формулировать как факты без источника. Если утверждение опирается на исследование — назвать его. Если нет — смягчить формулировку («как правило», «часто», «по наблюдениям»).

## 6. Структура статьи (сверху вниз)

### `<head>`

```html
<title>[Заголовок] | КУ</title>
<meta name="description" content="[Прямой ответ + цифра. До 160 символов.]">
<meta name="keywords" content="[5–10 ключей через запятую из semantic-core]">
<link rel="canonical" href="https://alkonorm.ru/articles/[хаб]/[слаг]/">
<link href="https://fonts.googleapis.com/css2?family=Montserrat:wght@300;400;500;600;700&family=Jost:wght@300;400;500;600&display=swap" rel="stylesheet">
<link rel="stylesheet" href="../../../assets/article.css">
```

JSON-LD: всегда `BreadcrumbList` (3 уровня). Если есть FAQ — добавить `FAQPage`.

### H1 + лид

- H1 = точная форма основного запроса из контент-плана.
- Лид: 2–4 предложения. Первое = прямой ответ с цифрой (из `docs/facts.md`). Без вводных слов.

### Блок быстрого ответа (`.quick-answer`)

Когда H1 — вопрос с цифрой или списком. Яндекс показывает в расширенном сниппете.

### Основной текст

- 2–4 H2. H2 = точный ключ из semantic-core (не «Общая информация»).
- 700–1500 слов. Один абзац — одна мысль (4–5 строк).
- H1 дословно не повторять более 2–3 раз.

### CTA-блок (обязателен)

Расположение: перед `<section class="related-section">`.

```html
<div class="cta-box">
    <h2>[Заголовок по теме]</h2>
    <p>[1–2 предложения о форуме]</p>
    <a href="https://kuforum.ru" target="_blank" rel="noopener" class="btn-primary">Перейти на форум →</a>
</div>
```

Запрещены классы `cta-block` и `btn-ku`.

### FAQ-блок

3–5 вопросов = поисковые запросы из semantic-core. Ответ самодостаточен (не «смотрите выше»). Дублировать в JSON-LD FAQPage.

### «Читать также»

3–5 ссылок на статьи того же хаба. Не более 1–2 из других хабов.

## 7. CSS — правила

**Никогда не добавлять `<style>` блок.**

| Тип файла | CSS |
|-----------|-----|
| Статья `articles/[хаб]/[слаг]/index.html` | только `article.css` (`../../../assets/article.css`) |
| Хаб `articles/[хаб]/index.html` | `article.css` + `hub.css` (`../../assets/`) |

**Классы `<body>` для хаб-страниц:**

| Хаб | Класс |
|-----|-------|
| `skolko-mozhno-pit` | *(без класса)* |
| `kak-menshe-pit` | `class="hub-narrow hub-green"` |
| `pochemu-hochetsya-alkogolya` | `class="hub-narrow hub-pink"` |
| `uznaki-alkogolizma` | `class="hub-narrow hub-orange"` |
